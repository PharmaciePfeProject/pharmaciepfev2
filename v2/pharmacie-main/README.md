# Pharmacy Platform

Full-stack pharmacy operations platform built for TRANSTU-style workflows, with Oracle-backed operational modules, RBAC-protected access, and Power BI report navigation.

## Stack

- Frontend: React, Vite, TypeScript, Tailwind CSS
- Backend: Node.js, Express
- Database: Oracle
- Auth: JWT
- Analytics access: Power BI links and optional embeds

## Main modules

- Authentication and registration
- RBAC with stable roles and permissions
- Admin user management
- Products
- Stock
- Stock lots
- Stock movements
- Distributions
- Inventories
- External orders
- Internal orders
- Receptions
- Internal deliveries
- BI and analytics access

## Run the backend

```bash
cd backend
npm install
npm run dev
```

Backend default URL:

```txt
http://localhost:4000
```

## Run the frontend

```bash
cd frontend
npm install
npm run dev
```

Frontend default URL:

```txt
http://localhost:5173
```

## Required environment variables

### Backend

Create `backend/.env` with at least:

```env
ORACLE_USER=
ORACLE_PASSWORD=
ORACLE_CONNECT_STRING=
ORACLE_SCHEMA=
JWT_SECRET=
```

`ORACLE_SCHEMA` can usually match the Oracle user schema used by the application.

### Frontend

Power BI dashboards are configured separately through public URLs in the frontend environment:

```env
VITE_POWERBI_STOCK_REPORT=
VITE_POWERBI_CONSUMPTION_REPORT=
VITE_POWERBI_DISTRIBUTION_REPORT=
VITE_POWERBI_MOVEMENTS_REPORT=
VITE_POWERBI_INVENTORY_REPORT=
```

Only use public embed URLs or public report links here. Do not place secrets in the frontend.

## Notes

- The application uses the existing Oracle operational schema as-is.
- Power BI dashboards are not generated by the app; the app only provides navigation and optional embedding.
- Public registration creates a standard user role only. Admin and advanced roles are assigned inside the platform.
